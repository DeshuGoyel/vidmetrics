                  6. YOUTUBE API ENGINE (lib/youtube.ts)
══════════════════════════════════════════════════════════════════

const YT = 'https://www.googleapis.com/youtube/v3'
const KEY = process.env.YOUTUBE_API_KEY

Build these functions:

─── parseChannelInput(input: string): { param: string; value: string } ───
Handle all YouTube URL formats:
  https://youtube.com/@handle       → { param: 'forHandle',  value: '@handle' }
  https://youtube.com/channel/UCxxx → { param: 'id',         value: 'UCxxx' }
  https://youtube.com/c/custom      → { param: 'forUsername', value: 'custom' }
  https://youtube.com/user/legacy   → { param: 'forUsername', value: 'legacy' }
  Plain @handle                     → { param: 'forHandle',  value: '@handle' }
  Plain UCxxx                       → { param: 'id',         value: 'UCxxx' }
Throw { code: 'INVALID_URL' } if none match.

─── fetchChannel(input: string): Promise<ChannelData> ────────────────────
Call: GET /channels?part=snippet,statistics,contentDetails&{param}={value}&key={KEY}
Map response.items[0] to ChannelData interface.
Throw { code: 'CHANNEL_NOT_FOUND' } if items is empty.
Throw { code: 'QUOTA_EXCEEDED' } if HTTP 403.
Throw { code: 'API_ERROR', message } for other errors.

─── fetchVideoIds(playlistId: string, max = 50): Promise<string[]> ───────
Call: GET /playlistItems?part=contentDetails&playlistId={id}&maxResults=50&key={KEY}
Return array of videoIds from items[].contentDetails.videoId
Handle pagination token for up to 50 results (single page is fine for MVP).

─── fetchVideoDetails(videoIds: string[]): Promise<RawVideo[]> ───────────
Chunk videoIds into arrays of 50.
For each chunk: GET /videos?part=snippet,statistics,contentDetails&id={ids}&key={KEY}
Flatten all results.
Map each item to VideoData interface, computing:
  - durationSeconds: parse ISO 8601 duration (PT8M23S → 503)
  - engagementRate: (likeCount + commentCount) / viewCount * 100
  - isRecent: publishedAt within last 30 days
  - youtubeUrl: https://youtube.com/watch?v={id}
  - keywords: extract top 3 meaningful words from title (exclude stop words)

─── enrichVideos(videos: VideoData[]): VideoData[] ───────────────────────
After fetching, add derived fields:
  - relativeViewScore: (viewCount / maxViewCount) * 100
  - isTrending: isRecent && viewCount > (avgViewCount * 1.5)
  - isTopAllTime: index === 0 after sorting by viewCount
  - engagementTier: 
      engagementRate > 5 → 'high'
      engagementRate > 2 → 'normal'
      else → 'low'
  - publishMonth: format(parseISO(publishedAt), 'MMM yyyy')

export function calculateHealthScore(channel: ChannelData, videos: VideoData[]): number
  Score 0-100 from 5 weighted components:
  1. Engagement rate (30%): avgEngagement / 5 * 100 (capped at 100)
  2. Upload consistency (20%): 1 if posts at least monthly, scale down
  3. View/subscriber ratio (20%): avgViews / subscriberCount * 100 (capped)
  4. Recent activity (15%): % of videos published in last 90 days
  5. View growth trend (15%): compare first-half vs second-half avg views
  Return Math.min(100, Math.round(weighted sum))

export function getTopVideos(videos: VideoData[], n = 10): VideoData[]
  Sort by viewCount desc, return top n

export function getRecentVideos(videos: VideoData[], days = 30): VideoData[]
  Filter publishedAt within last {days} days

export function getAvgMetric(videos: VideoData[], key: keyof VideoData): number
  Return mean of numeric field across all videos

export function getChartData(videos: VideoData[], metric: ChartMetric): ChartPoint[]
  Sort videos by publishedAt ascending
  Map to { date: publishMonth, value: metric-value, title: truncated-title }
  Aggregate by month if > 30 videos (sum views per month)

export function getUploadCadenceData(videos: VideoData[]): CadencePoint[]
  Group videos by publishMonth
  Return [{ month: string, uploads: number, avgViews: number }]

────── MODE 2: Algorithmic Insights ─────────────────────────────

Always works. Build 4 insights from pure data math:

INSIGHT 1 — CONTENT STRATEGY (type: 'strategy', icon: '🎯'):
  - Tokenize all titles, remove stop words, count word frequency
  - stopWords = ['the','a','an','in','on','at','how','to','my','your','i',
                 'and','or','of','this','is','are','was','for','with','what']
  - Find top 3 keywords (e.g. "review", "tutorial", "reaction")
  - Count videos containing each keyword → percentage
  - title: "Content Strategy Pattern"
  - body: "'{top keyword}' content makes up {X}% of videos,

  - stat: `${X}% keyword concentration`

INSIGHT 2 — UPLOAD TIMING (type: 'timing', icon: '📅'):
  - Map each video's publishedAt to day-of-week (0=Sun...6=Sat)
  - Count uploads per day → find peak day
  - Calculate avg viewCount for videos on peak day vs others
  - Compute avg days between uploads (total days span / video count)
  - title: "Optimal Upload Pattern"
  - body: "Posts every {X.X} days. {DayName} uploads average

INSIGHT 3 — TOP VIDEO FORMULA (type: 'formula', icon: '🔥'):

  - Analyze: avg duration, title length, common prefix patterns

  - title: "Top Video Formula"
  - body: "Best videos average {X} min runtime with {Y}-word titles.

  - stat: `${X} min avg runtime`

INSIGHT 4 — GROWTH TRAJECTORY (type: 'trajectory', icon: '📈'):

GET /api/analyze?url={channelUrl}

Logic:

2. If YOUTUBE_API_KEY is missing or url === 'demo':

3. Try/catch the full pipeline:
   a. channel = await fetchChannel(url)
   b. videoIds = await fetchVideoIds(channel.uploadPlaylistId, 50)
   c. rawVideos = await fetchVideoDetails(videoIds)
   d. videos = enrichVideos(rawVideos)
   e. channel.avgEngagementRate = avg of videos[].engagementRate
   f. channel.healthScore = calculateHealthScore(channel, videos)
   g. insights = await generateInsights(channel, videos)
   h. Return NextResponse.json({ channel, videos, insights, 
                                  fetchedAt: new Date().toISOString(),
                                  isDemo: false }, 
                               { headers: { 'Cache-Control': 's-maxage=3600' } })

   CHANNEL_NOT_FOUND → 404, "We couldn't find that channel. Check the URL."
   QUOTA_EXCEEDED    → 429, "YouTube API daily limit reached. Try again tomorrow."
   INVALID_URL       → 400, "Please enter a valid YouTube channel URL."

              24. UTILITY FUNCTIONS (lib/utils.ts)

export function cn(...inputs: ClassValue[]): string
  → clsx + tailwind-merge

export function formatNumber(n: number): string
  → 1234567 → "1.2M" | 45300 → "45.3K" | 999 → "999"

export function formatViews(n: number): string
  → same but appends " views" for < 1000

export function parseDurationToSeconds(iso: string): number
  → "PT1H8M23S" → 4103 | "PT8M23S" → 503 | "PT45S" → 45

export function formatDuration(seconds: number): string
  → 4103 → "1:08:23" | 503 → "8:23" | 45 → "0:45"

export function timeAgo(dateString: string): string
  → Uses date-fns formatDistanceToNow + "ago" suffix
  → "just now" for < 1 minute

export function getEngagementColor(rate: number): string
  → rate > 5: "text-green" | > 2: "text-cyan" | else: "text-text-muted"

export function extractKeywords(title: string): string[]
  → lowercase, split words, filter stop words, return top 3 by length/freq

export function truncate(str: string, max: number): string
  → str.length > max ? str.slice(0, max) + '...' : str

export function isValidYouTubeUrl(input: string): boolean
  → Regex: matches youtube.com/*, youtu.be/*, or starts with @

══════════════════════════════════════════════════════════════════

URL input errors (inline, below input):
  Not a URL at all: "Enter a YouTube channel URL to get started"
  Wrong domain: "This doesn't look like a YouTube URL"
  Missing handle: "Include the channel handle, e.g. youtube.com/@channel"

API errors (full-page error state with retry):
  CHANNEL_NOT_FOUND: "Channel not found. Double-check the URL and try again."
  QUOTA_EXCEEDED: "YouTube API limit reached for today. Try demo mode instead."
  API_ERROR: "Couldn't connect to YouTube. Please try again in a moment."
  NETWORK_ERROR: "Network error. Check your connection and retry."

Each error state:
  - On-brand SVG icon (geometric, no emojis for main errors)
  - Clear heading + helpful description
  - "Try Again" button → calls store.analyze(inputUrl)
  - "Use Demo Mode" secondary button → calls store.analyze('demo')
